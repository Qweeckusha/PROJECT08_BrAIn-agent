# Документация `ingest.py` (Ядро записи)

Файл отвечает за **приём сырого текста**, **структурирование через LLM**, **поиск дубликатов/связей в FAISS** и **инкрементальное обновление** локальной базы знаний (`brain.json` + `faiss_index/`).

## Архитектура пайплайна
```
Текст → LLM-парсинг (JSON) → FAISS-поиск → Оценка сходства → LLM-валидация (Если дубликат, возможно дополонение) → Сохранение в базу знаний
```


## Конфигурация и инициализация
| Параметр | Значение | Назначение                               |
|----------|----------|------------------------------------------|
| `GENERATION_MODEL` | `qwen/qwen3-4b-2507` | Основная модель для парсинга и валидации |
| `EMBEDDING_MODEL` | `all-mpnet-base-v2` | Векторизатор (768 dim, нормализация L2)  |
| `FAISS_INDEX` | `./faiss_index/` | Хранение faiss индекса                   |
| `BRAIN_FILE` | `./brain.json` | Хранилище записей (JSON)                 |


## Промпты и Chains
### `PARSE_PROMPT`
Превращает произвольный текст в строгую JSON-схему:
`{text, topic, subtopic, tags, level, status}`
*Цель:* Извлечь смысл без markdown-обёрток. При отсутствии данных возвращает `{}`.

### `VALIDATE_PROMPT`
Классификатор новизны. Сравнивает `{new_text}` с `{context}` (список существующих записей).
*Цель:* Увеличение точности за счёт дополнительного вызова LLM для обнаружения сложных связей, которые эмбеддер не увидит
*Выход:* `duplicate` | `complement` | `new`


### Chains
```python
parse_chain = PARSE_PROMPT | llm | output_parser
validate_chain = VALIDATE_PROMPT | llm | output_parser
```


## Вспомогательные функции
| Функция                                  | Логика                                                                                                                                         |
|------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------|
| `load_brain()` / `save_brain()`          | Загрузка/сохранение для `brain.json`                                                                                                           |
| `rebuild_faiss_index()`                  | Полная пересборка FAISS. Векторизует `brain.json` вручную, применяет `faiss.normalize_L2`, синхронизирует `docstore` и `index_to_docstore_id`. |
| **`find_similar_lc(query, db)`**         | Поиск `top_k` чанков с фильтром по `threshold=0.75`. Возвращает отсортированный список `{id, similarity, text}`. ОЧЕНЬ ВАЖНАЯ ФУНКЦИЯ          |
| `decide_with_llm_lc(new_text, existing)` | Вызов LLM для финального вердикта, если семантическое сходство > 0.90.                                                                         |
| `save_knowledge_lc(parsed, db)`          | Сохраняет запись в `brain.json`, генерирует `uuid`, добавляет вектор в FAISS (`db.add_texts`) и сохраняет индекс на диск.                      |


## Контракт API (`process_ingest`)
```python
def process_ingest(text: str) -> dict:
```
**Вход:** Сырой текст пользователя.  
**Выход:** JSON-словарь с полями `status`, `message`, `data`.

### Возвращаемые статусы
| Status | Условие | `data` |
|--------|---------|--------|
| `error` | Пустой ввод, крах парсинга JSON, исключение | `{"message": "..."}` |
| `duplicate` | Сходство > 0.90 + LLM подтвердил дубликат | `{"match": "..."}` |
| `success` | Запись сохранена (`new` или `complement`) | `{"id", "topic", "decision", "related_to"}` |

### Логика принятия решений
1. `score > 0.90` → Вызов `validate_chain` → `duplicate` / `complement` / `new`
2. `0.75 < score ≤ 0.90` → Автоматически `complement` (привязка через `related_ids`)
3. `score ≤ 0.75` или пусто → `new`
